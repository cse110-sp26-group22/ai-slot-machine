# App & Code Quality Evaluation Rubric

## 1. Functionality (20%)
**Does the app actually work as intended?**

| Level | Criteria |
|------|--------|
| Excellent (5) | All features work correctly, edge cases handled, no crashes |
| Good (4) | Core features work, minor bugs present |
| Average (3) | Some features incomplete or buggy |
| Poor (2) | Major functionality missing or broken |
| Very Poor (1) | App does not function |

---

## 2. User Experience (UX/UI) (15%)
**Is the app easy and pleasant to use?**

| Level | Criteria |
|------|--------|
| 5 | Intuitive navigation, consistent design, responsive UI |
| 4 | Mostly easy to use, small inconsistencies |
| 3 | Some confusion or clutter |
| 2 | Hard to navigate, poor layout |
| 1 | Frustrating or unusable |

---

## 3. Performance & Efficiency (10%)
**How fast and resource-efficient is the app?**

| Level | Criteria |
|------|--------|
| 5 | Fast load times, optimized algorithms |
| 4 | Minor slowdowns |
| 3 | Noticeable lag in some areas |
| 2 | Frequent slowdowns |
| 1 | Very slow / freezes |

---

## 4. Security & Reliability (10%)
**Is the app safe and stable?**

| Level | Criteria |
|------|--------|
| 5 | Input validation, secure handling, no crashes |
| 4 | Minor security gaps |
| 3 | Some vulnerabilities |
| 2 | Major risks (e.g., no validation) |
| 1 | Unsafe / highly unstable |

---

# Code Quality Section

## 5. Readability & Style (10%)
**Is the code easy to read and understand?**

| Level | Criteria |
|------|--------|
| 5 | Clean, consistent naming, follows style guides |
| 4 | Mostly clean |
| 3 | Some messy sections |
| 2 | Hard to follow |
| 1 | Unreadable |

---

## 6. Structure & Organization (10%)
**Is the code well-architected?**

| Level | Criteria |
|------|--------|
| 5 | Modular, clear separation of concerns |
| 4 | Mostly organized |
| 3 | Some tightly coupled code |
| 2 | Poor structure |
| 1 | Spaghetti code |

---

## 7. Maintainability & Scalability (5%)
**How easy is it to extend or modify?**

| Level | Criteria |
|------|--------|
| 5 | Easy to extend, scalable design |
| 4 | Minor refactoring needed |
| 3 | Moderate effort required |
| 2 | Hard to modify |
| 1 | Very brittle |

---

# Scoring

- Each category: score from **1–5**
- Multiply by weight
- Final score out of **100**

### Example

```
Functionality: 4 × 20 = 80
UX: 5 × 15 = 75
Performance: 3 × 10 = 30

Total = (Sum of all scores ÷ 5) = Final Score / 100
```
